Architecture Review — Timestamp Navigation

mtgnews · 2026-06-11

Module Seam Deep module Leakage

Bug Context

Clicking a timestamp in the Signal List chat panel navigates to Signal Detail but the transcript pane stays closed, no scroll occurs, and no highlight appears. The identical click inside the Signal Detail chat panel works perfectly: split view opens, smooth scroll centers the line, yellow flash highlights it.

Problem: Two shallow formatter modules produce incompatible link formats. CitationFormatter omits data-timestamp, making its links invisible to the click interception seam in signal-detail.ejs. The format divergence is the root cause of the bug.

Solution: Collapse both formatters into a single deep module: TimestampLink.format(text, signalMap?, samePage?). Always emits uniform links with data-timestamp, data-video-id, and full-URL href. The consumer's click handler decides whether to intercept (same page) or navigate (cross-page).

Wins:

  • Locality: link format lives in one module, not two
  • Leverage: one interface serves both single and multi-signal chat
  • Deletion test: deleting CitationFormatter scatters citation logic into callers. Keeping it here concentrates it.
  • Tests: one test surface for all timestamp link formats

Problem: Timestamp click interception is hardcoded inside signal-detail.ejs Alpine init(). The signals list page has no equivalent handler. Click logic is scattered across EJS templates instead of concentrated in a framework-agnostic JS module.

Solution: Extract into views/scripts/timestamp-nav.js: a deep module with TimestampNav.init({ samePage, scrollCallback }). Signal Detail passes samePage: true (intercept + scroll). Signals List passes samePage: false (navigate to signal detail with hash intent).

Wins:

  • Locality: click behavior in one module, not scattered across EJS
  • Leverage: same handler works for both pages via config
  • Two adapters justify the seam: same-page intercept vs cross-page navigate
  • E2E testable through the module interface, not DOM probing
Existing pattern: This follows the exact same pattern as scope-source.js — framework-agnostic JS module loaded before Alpine, consumed by both pages. The precedent is already set in this codebase.

Problem: When navigating from the Signal List via a timestamp link (/signals/abc#t-45000), the detail page loads with default viewState: 'summary'. The transcript pane is hidden, so even browser-native hash scrolling targets an invisible element. No visible behavior occurs.

Solution: Add a single 3-line check in init(): read window.location.hash, match against /^#t-(\d+)$/, and if found, call the existing handleSummaryPillClick(ms). Reuses all scroll/center/highlight logic.

Wins:

  • Locality: hash processing lives where scroll logic already exists
  • Zero new interfaces — calls existing handleSummaryPillClick()
  • Closes the navigation gap from cross-page timestamp clicks

Top Recommendation — Tackle B first, then A, then C

B (TimestampNav module) is the highest-leverage change. It creates a real seam with two adapters (same-page intercept vs cross-page navigate), follows the existing scope-source.js pattern, and immediately fixes the signals list page by adding click interception. Without B, the other candidates are incomplete: uniform links (A) still need something to catch them, and hash processing (C) only helps after navigation happens.

A (TimestampLink module) then eliminates the format divergence at source — one deep module instead of two shallow formatters. This is the structural fix that prevents future regressions.

C (hash processing) is a 3-line addition in signal-detail.ejs init(). Minimal risk, closes the last gap. Apply after B so navigation actually delivers hash fragments worth processing.
